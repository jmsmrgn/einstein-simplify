# Before / After: Kubernetes

## Before

> "Kubernetes is an open-source container orchestration platform
> that automates the deployment, scaling, and management of
> containerized applications. It groups containers that make up an
> application into logical units for easy management and discovery."

## After

> "Imagine each piece of your application — the login service, the
> payment processor, the database — lives in its own isolated box.
> Those boxes are called containers. Now imagine you have hundreds
> of those boxes running across dozens of servers, and you need to
> make sure they all stay running, can find each other, and scale up
> when traffic spikes. Doing that by hand is a full-time job.
> Kubernetes is the system that does it automatically. You tell it
> what you want running and it keeps it that way."

## Changes

| #   | Change                                                                                     | Principle                         |
| --- | ------------------------------------------------------------------------------------------ | --------------------------------- |
| 1   | "container orchestration platform" replaced with isolated box scenario                     | Analogy before description        |
| 2   | "containers" introduced after the concept (isolated boxes) exists                          | Concept before label              |
| 3   | "logical units for easy management and discovery" replaced with concrete scenario          | Replace jargon, don't define it   |
| 4   | Value stated last in one plain sentence: "you tell it what you want, it keeps it that way" | Teachable moment given full space |
